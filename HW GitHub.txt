XML
 21. Создать внешний репозиторий c названием XML - сделан в веб интерфейсе
-------------------------------------------------
-------------------------------------------------
 22. Клонировать репозиторий XML на локальный компьютер - git clone https://github.com/VladimirKolosov/XML.git
-------------------------------------------------
$ ls
JSON/  Terminal-Linux/  XML/  tr4nk/

-------------------------------------------------
 23. Внутри локального XML создать файл “new.xml” - nano new.xml
-------------------------------------------------
$ ls
README.md  new.xml
-------------------------------------------------
 24. Добавить файл под гит - git add new.xml
-------------------------------------------------
-------------------------------------------------
 25. Закоммитить файл - git commit -m"Новый xml"
-------------------------------------------------
-------------------------------------------------
 26. Отправить файл на внешний GitHub репозиторий - git push
-------------------------------------------------
-------------------------------------------------
 27. Отредактировать содержание файла “new.xml” - nano new.xml
-------------------------------------------------
$ cat new.xml
<?xml version="1.0" encoding="windows-1251"?>
<about me>
   <Name>Kolosov Vladimir Olegovich</Name>
   <Birthday>28.08.1994</Birthdayr>
   <Pets>2</Pets>
   <Future salary> No limits</Future salary>
</about me>
-------------------------------------------------
 28. Отправить изменения на внешний репозиторий - git commit -am"changed file new.xml", git push. В веб интерфейсе появился изменненый файл
-------------------------------------------------
-------------------------------------------------
 29. Создать файл preferences.xml - nano preferences.xml
-------------------------------------------------
$ ls
README.md  new.xml  preferences.xml
-------------------------------------------------
 30. В файл preferences.xml добавить информацию о своих предпочтениях - nano preferences.xml
-------------------------------------------------
 cat preferences.xml
<?xml version="1.0" encoding="UTF-8" ?>
<preferences>
                <favoritefilm>Avengers</favoritefilm>
                <favoritefood>Pasta</favoritefood>
                <favoritetimeofyear>Summer</favoritetimeofyear>
                <countryforvisit>Greece</countryforvisit>
</preferences>
-------------------------------------------------
 31. Создать файл sklls.xml добавить информацию о скиллах которые будут изучены на курсе в формате XML - nano skills.xml
-------------------------------------------------
$ cat skills.xml
<?xml version="1.0" encoding="UTF-8" ?>
<skills>
                <mobiletesting>Мобильное тестирование</mobiletesting>
                <vpn>Как работает VPN, для чего нужен и как использовать</vpn>
                <androidstudio>Сборка приложений на Androit</androidstudio>
                <adb>Управление андройд девайсами</adb>
</skills>
-------------------------------------------------
 32. Сделать коммит в одну строку - git commit -m "new files"
-------------------------------------------------
-------------------------------------------------
 33. Отправить сразу 2 файла на внешний репозиторий - git push
-------------------------------------------------
-------------------------------------------------
 34. На веб интерфейсе создать файл bug_report.xml - в репозитории XML нажимаем кнопку Add file и выбираем Create new file(задаем нужное имя файла)
-------------------------------------------------
-------------------------------------------------
 35. Сделать Commit changes (сохранить) изменения на веб интерфейсе - открыл cозданный файл для редактирования, ввел информацию и нажал кнопку commit changes
-------------------------------------------------
-------------------------------------------------
 36. На веб интерфейсе модифицировать файл bug_report.xml, добавить баг репорт в формате XML - открыл созданный файл и нажал кнопку Edit this file
-------------------------------------------------
-------------------------------------------------
 37. Сделать Commit changes (сохранить) изменения на веб интерфейсе - после ввода информации нажал на кнопку Сommit changes, в файле появился введенный текст
-------------------------------------------------
-------------------------------------------------
 38. Синхронизировать внешний и локальный репозиторий XML - gip pull

$ ls
